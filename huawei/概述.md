## 渠道特点

### 商品类型

华为 IAP 支持以下类型的虚拟商品：

1. **消耗型商品（Consumable）**：使用一次后即消耗掉，需要再次购买。例如：游戏货币、游戏道具等。
2. **非消耗型商品（Non-Consumable）**：一次性购买，永久拥有，无需消耗。例如：游戏额外关卡、无时限高级会员等。
3. **订阅型商品（Subscription）**：用户购买后在一段时间内允许访问增值功能或内容，周期结束后自动续期。例如：视频月度会员等。

### 主要特点

1. **接入便捷，沙盒测试方便**
   - 完善的接入流程指导和配套沙盒环境，帮助开发者快速接入华为 IAP。

2. **全球发布，本地化语言和货币**
   - 支持根据用户帐号服务地所在国家/地区，自动展示本地化语言和货币价格。
   - 发布一个应用，即可在华为 IAP 覆盖的所有国家/地区推广。

3. **聚合多种支付通道，提供全球支付服务**
   - 支持银行卡（国内主流银行、Visa、Mastercard 等）、支付宝、微信支付、运营商话费、华为虚拟币（花币）等多种支付方式。
   - 话费支付支持多个海外大型运营商。
   - 支持绑定银行卡、手机号码到华为帐号，支付时只需输入华为支付密码即可完成支付。

4. **安全可靠，监控有保障**
   - 采用业界标准的 TLS 协议加密传输数据。
   - 通过 RSA 算法签名保护用户消息，防止截取和篡改。
   - 数据后台采用加密和匿名化方式存储。
   - 后台服务器采用集群部署和冗余设计，7×24 小时实时监控系统运行。

### API 接口分类

| 分类 | 平台 | 说明 |
|------|------|------|
| 订单、订阅服务 | Android（Java）| 查询商品信息、创建订单、查询已购商品订单信息、订阅页面跳转等 |
| 订单、订阅服务 | HarmonyOS（JavaScript）| 查询商品信息、创建订单、查询已购商品订单信息等 |
| 订单、订阅服务 | Server（HTTPS）| 验证购买 Token、确认购买、订单查询、订阅管理等服务端接口 |
| VR 支付服务 | Android | 应用内支付、订单信息查询 |
| VR 支付服务 | Server（HTTPS）| 服务端回调（支付成功后，IAP 服务器向开发者配置的 URL 回调支付结果） |
