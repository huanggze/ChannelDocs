## 一、商品类型

| 商品类型 | 说明 | 示例 |
|--------|------|------|
| 消耗型商品 | 使用一次后即消耗掉，随使用减少，需要再次购买 | 游戏货币、游戏道具 |
| 非消耗型商品 | 一次性购买，永久拥有，无需消耗 | 额外游戏关卡、无时限高级会员 |
| 订阅型商品 | 周期性付费，周期结束后自动续期 | 视频月度会员、有时限高级会员 |

---

## 二、商品配置

在华为 AppGallery Connect 后台配置商品，主要字段：

- **商品 ID（productId）**：唯一标识一个商品，在应用内保持唯一
- **商品类型**：消耗型（0）/ 非消耗型（1）/ 订阅型（2）
- **货币与定价**：按用户所在国家/地区展示本地化价格，支持多地区定价配置
- **商品名称与描述**：本地化多语言配置

> 说明：华为 IAP 支持在所有覆盖国家/地区展示本地化语言和货币，只需配置一套商品即可全球销售。

## 三、支付流程

### 3.1 消耗型商品购买流程

1. 用户发起购买 → 应用客户端向 HMS Core（APK）发起购买请求，携带商品 ID、商品类型等信息
2. HMS Core（APK）生成订单，返回 IAP 收银台
3. 应用客户端启动 IAP 收银台
   - > 若使用 `createPurchaseIntentWithPrice` 接口发起支付，需使用 IAP 公钥对接口返回的 `paymentData` 数据进行签名校验
4. 用户完成支付后，HMS Core（APK）返回购买数据 `InAppPurchaseData`（含是否购买成功、订单 ID、商品 ID、购买 Token 等）及其签名数据
5. 应用客户端向应用服务器上报购买数据及签名数据
6. **（建议）** 应用服务器调用 `Order服务验证购买Token` 接口，向 IAP 服务器查询购买数据，进一步确认订单准确性
7. **（建议）** IAP 服务端返回购买数据及签名数据
8. 应用服务器使用 IAP 公钥验证购买数据签名；验签成功后**必须校验** `InAppPurchaseData` 中 productId、price、currency 等信息的一致性
9. 验证成功后，发放商品，**记录商品的发货状态（购买 Token + 商品 ID）**
   - > ⚠️ **注意**：务必保存已发货商品的购买 Token，后续即使消耗失败也可从服务器获取发货状态，避免重复发货
10. 应用服务器返回发货结果给应用客户端
11. **发货成功后**，应用客户端向 HMS Core（APK）发送 `consumeOwnedPurchase` 请求以消耗商品，通知 IAP 服务器更新发货状态（也可使用服务端 `Order服务确认购买` 接口替代）
    - > ⚠️ **注意**：消耗必须在发货成功后执行，否则可能出现服务器已消耗但应用未发货的异常场景
12. HMS Core（APK）向应用返回消耗结果；消耗成功后，IAP 服务器将商品重新设置为可购买状态

### 3.2 非消耗型商品购买流程

1. 用户发起购买 → 应用客户端向 HMS Core（APK）发起购买请求
2. HMS Core（APK）生成订单，返回 IAP 收银台
3. 应用客户端启动 IAP 收银台
4. 用户完成支付后，HMS Core（APK）返回购买数据 `InAppPurchaseData` 及签名数据
5. 应用客户端向应用服务器上报购买数据及签名数据
6. **（建议）** 应用服务器调用 `Order服务验证购买Token` 接口验证
7. **（建议）** IAP 服务端返回购买数据及签名数据
8. 应用服务器使用 IAP 公钥验证签名；验签成功后必须校验 productId、price、currency 等字段一致性
9. 验证成功后，通过 `purchaseState` 字段判断购买状态：`purchaseState=0` 表示已购买，发放商品

> 说明：非消耗型商品用户只需购买一次即可永久拥有，无需消耗步骤。

### 3.3 订阅型商品购买流程

**首次购买：**

1. 用户发起购买 → 应用客户端向 HMS Core（APK）发起 `createPurchaseIntent` 购买请求，携带商品 ID、商品类型等信息
2. HMS Core（APK）生成订单，返回 IAP 收银台
3. 应用客户端启动 IAP 收银台
4. 用户完成支付后，HMS Core（APK）返回购买数据 `InAppPurchaseData`（含是否购买成功、订阅 ID、商品 ID、购买 Token 等）及签名数据
5. 应用客户端向应用服务器上报购买数据及签名数据
6. **（建议）** 应用服务器调用 `Subscription服务验证购买Token` 接口，向 IAP 服务器查询购买数据，需携带购买 Token 和订阅 ID
7. **（建议）** IAP 服务端返回购买数据及签名数据
8. 应用服务器使用 IAP 公钥验证签名；验签成功后必须校验 productId、price、currency 等字段一致性
9. 验证成功后，向用户提供相应的商品服务

**持续提供服务（订阅有效期内）：**

1. 应用客户端向 HMS Core（APK）发送 `obtainOwnedPurchases` 请求，查询已购订阅型商品
2. HMS Core（APK）返回用户所有已订阅且正生效的商品购买数据 `InAppPurchaseData`（含订阅 ID、商品 ID、购买 Token 等）及签名数据
3. 应用客户端向应用服务器上报数据
4. 应用服务器使用 IAP 公钥验证签名，并校验 productId、price、currency 等字段一致性
5. 验证成功后，向用户提供相应的商品服务

## 四、服务端 API

华为 IAP 服务端提供以下核心 API（均需通过华为 OAuth2 获取 Access Token 后调用）：

| API 名称 | 用途 | 商品类型 |
|---------|------|---------|
| **Order服务验证购买Token** | 验证消耗型/非消耗型商品的购买 Token，获取购买数据及签名 | 消耗型、非消耗型 |
| **Subscription服务验证购买Token** | 验证订阅型商品的购买 Token，获取订阅购买数据及签名 | 订阅型 |
| **Order服务确认购买** | 服务端消耗商品（替代客户端 `consumeOwnedPurchase` 接口） | 消耗型 |
| **取消订阅** | 开发者主动取消用户订阅 | 订阅型 |
| **延迟结算** | 为订阅设置延迟结算日期 | 订阅型 |
| **返还订阅费用** | 为订阅返还费用 | 订阅型 |
| **撤销订阅** | 撤销用户的订阅 | 订阅型 |
| **已取消或者退费购买查询** | 查询已取消或退费的购买记录 | 全部 |
| **应用购买记录相关支付订单查询** | 查询应用购买记录关联的支付订单 | 全部 |
| **接收订单/订阅关键事件通知** | 服务端接收 IAP 推送的订单/订阅状态变更通知（被动回调） | 全部 |
| **接收延迟付款型支付关键事件通知** | 服务端接收延迟付款类型的关键事件通知 | 特定场景 |
| **对返回结果验签** | 对 IAP 接口返回数据进行签名验证 | 全部 |
| **（必要）验证InAppPurchaseData** | 必须校验 InAppPurchaseData 中的关键字段一致性 | 全部 |

### Order服务验证购买Token 说明

- **用途**：在完成一笔支付后，通过 purchaseToken 向华为 IAP 服务器查询购买数据，进一步确认订单准确性
- **适用范围**：仅针对消耗型商品和非消耗型商品
- **请求参数**：需携带 purchaseToken + productId
- **返回数据**：purchaseData（`InAppPurchaseData` JSON 字符串）+ dataSignature（签名）

### 被动回调（事件通知）说明

华为 IAP 支持服务端推送通知（Notification），当以下事件发生时，华为会向开发者配置的回调 URL 推送消息：
- 订阅续期成功/失败
- 用户取消订阅
- 订阅到期
- 退款
- 订阅恢复
- 延迟付款相关事件

开发者需在 AppGallery Connect 后台配置回调 URL，并在接收到通知后返回 200 响应。

## 五、资损防控关键字段

验签成功后必须校验以下字段，防止资损：

| 字段名 | 说明 | 必查原因 |
|-------|------|---------|
| **productId** | 商品 ID | 防止用低价商品的 Token 冒充高价商品 |
| **price** | 商品价格 | 确认实际支付金额与预期一致 |
| **currency** | 货币类型 | 防止货币单位不一致导致价格判断错误 |
| **purchaseState** | 购买状态（0=已购买，1=已取消，2=退款中） | 确认商品确实已完成支付 |
| **purchaseToken** | 购买 Token | 唯一标识用户与商品的对应关系 |
| **developerPayload** | 开发者自定义透传参数 | 关联应用帐号，防止权益发放给错误帐号 |
| **orderId** | 订单 ID | 幂等控制，防止重复发货 |
| **purchaseTime** | 购买时间（毫秒时间戳） | 判断订单有效性，防止重放攻击 |
| **kind** | 商品类型（0=消耗型，1=非消耗型，2=订阅型） | 确认商品类型与业务预期一致 |

### developerPayload 使用建议

- 使用应用帐号相关信息作为 developerPayload，推荐通过单向哈希或加密混淆后生成标识符
- **不要使用业务订单号**作为 developerPayload 来确定订阅归属（订单失效或丢失时会无法确定订阅归属）
- 订阅场景下 developerPayload 会与订阅绑定，整个生命周期保持不变（恢复订阅时可更新）
